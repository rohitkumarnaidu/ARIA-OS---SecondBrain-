## 1. Who We Are and Scope

**Second Brain OS** (operating as **ARIA OS**) is a personal AI productivity system developed and operated by a single developer (a BTech CSE student based in India). This Privacy Policy explains how we collect, use, store, and protect your personal data when you use the ARIA OS application (the "Service").

This policy applies to all users of the Service, regardless of location. We have designed this policy to comply with the **General Data Protection Regulation (GDPR)** (EU) 2016/679, the **Digital Personal Data Protection Act (DPDP Act) 2023** (India), and other applicable privacy laws. Where local law provides stronger protections, those protections apply.

**Data Controller:** The individual developer who operates the Service. For the purposes of GDPR Article 4(7), the developer is the data controller.

**Important note:** This is a single-developer project. While we take data protection seriously, the Service does not have a registered legal entity, a physical office, or dedicated legal counsel. We process personal data transparently and provide direct access to the developer for any privacy concerns.

## 11. Security Measures

We implement industry-standard technical and organizational measures to protect your data. Here is what we do:

### 11.1 Technical Measures

| Measure | Implementation |
|---|---|
| **Encryption in transit** | TLS 1.3 on all connections (enforced by Vercel + Railway edge) |
| **Encryption at rest** | AES-256 encryption (Supabase PostgreSQL) |

| **Authorization** | Row-Level Security (RLS) on every database table â€” your data is isolated from every other user |
| **Input validation** | Pydantic schemas on all API endpoints â€” malformed requests are rejected |
| **Rate limiting** | 100 requests per minute per IP; AI chat limited to 30 requests per minute |
| **SQL injection protection** | Parameterized queries via Supabase SDK â€” raw SQL queries not accepted |
| **XSS protection** | Input sanitization middleware on all user-submitted content |
| **CSRF protection** | Dedicated CSRF middleware |
| **Audit trail** | Structured JSON logging with unique request IDs for every API call |
| **DDoS protection** | Vercel WAF (Web Application Firewall) + Railway edge protection |
| **Secrets management** | All credentials stored in environment variables â€” never in code or version control |
| **Dependency scanning** | Automated vulnerability scanning via `npm audit`, `pip-audit`, and `safety` in CI pipeline |

| Measure | Description |

| **Penetration testing** | Quarterly SAST (OWASP ZAP) + DAST (custom attack scripts) |

### 11.3 What We Cannot Guarantee

While we take security seriously, no system is 100% secure. We cannot guarantee absolute security. We use third-party infrastructure (Supabase, Vercel, Railway) that has its own security measures, and we rely on those providers' security postures. We encourage you to:
