## Task Manager

### Smart Creation
- Type task description in natural language
- AI automatically assigns: priority (high/medium/low), category (study/build/health/admin), estimated time
- AI estimates based on historical accuracy for similar tasks

### Auto-Reschedule
- Background cron every 15 minutes (pg_cron or APScheduler)
- Detects overdue tasks (due_date < now(), status not done/archived)
- Moves to next available slot (now + 2 hours default)
- Increments missed_count with each reschedule

### Missed Task Policy
- Zero missed policy — non-negotiable
- Every task: done, rescheduled, or explicitly dropped
- No silent failures — user always knows status
- Escalation ladder: push → email at missed_count >= 2 → SMS at missed_count >= 3

### Subtask AI Breakdown
- Ask ARIA to break any complex task
- Creates manageable subtasks automatically
- Subtasks inherit parent's priority and category
- Subtask completion updates parent progress

### Dependencies
- Link tasks that must happen in sequence
- Visual dependency graph (arrow connections)
- Auto-blocked if dependency not complete
- Reschedule cascading on dependency shift

### Recurring Tasks
- Daily, weekly, or custom frequency (cron expression)
- Smart rescheduling based on completion history
- Skip weekends option
- Auto-create next occurrence on completion

### Sleep-Aware Priority
- Tasks reprioritized based on sleep score each morning
- Low sleep (<50) = lighter tasks (admin, review, planning) prioritized
- High sleep (>70) = full schedule with deep work
- Medium sleep (50-69) = moderate adjustment, heavy tasks moved to afternoon

### Escalation
- Push notification immediately after deadline
- Email via Resend after 30 minutes (missed_count >= 2)
- SMS via Twilio after 1 hour for critical items (missed_count >= 3)

### Task Filters
- By status: todo / in_progress / done / missed / archived
- By priority: high / medium / low
- By category: study / build / health / admin
- By date: today / this week / overdue / all
- Free text search across titles and descriptions

### Mandatory Deadline
- No course accepted without target completion date

- Always visible on course card
- Free text — "why did I sign up for this?"

- Course creates daily study task automatically in task manager
- Task linked to course ID

- Minutes set to daily_minutes_target

### Spaced Repetition
- Review at 1, 3, 7, 14, 30 days after studying
- Integrated with task manager as review tasks

### Behind-Schedule Alert
- ARIA warns if pace is insufficient 2 weeks before deadline
- Recalculates daily minute target
- Option to extend deadline or increase daily minutes
- Visual indicator on course card (green=on track, yellow=warning, red=behind)

### Course Progress Nudge (6 PM)
- Daily check: did user study target minutes today?
- If not: "You haven't studied [course]. [X]min before bed to stay on track."
- If behind schedule: recalculates and alerts new daily target

### Course Categories
- Active / Paused / Completed / Dropped
- Completion date recorded when moved to completed
- Total study hours tracked per course

### Visual Roadmap Builder
- Drag-and-drop canvas (React Flow)
- 8 node types: skill, milestone, project, phase, exam_topic, goal, resource, custom
- Works on phone and desktop
- Node colors by type (blue/green/orange/purple/red/gold/grey)

4. PDF upload — syllabus, study guide, pdf-parse + Tesseract OCR

### 8 Roadmap Types
- Career / Skills
- Business Launch
- Exam Prep
- Study / Learning
- Project
- Health
- Financial
- Custom

### AI Timing
- Evidence-based time estimates from industry benchmarks
- Three sliders: hours/day (1-8), days/week (1-7), intensity (0.7x-1.3x)
- Real-time recalculation of all completion dates on slider change
- Projected end date shown at top of canvas

- Detects missed milestones
- Recalculates entire downstream timeline
- Notifies user of new projected completion date
- Option to adjust sliders or accept new timeline

- Works backwards from exam date

- What-if simulations

### Task Generation
- Every active roadmap node creates tasks in task manager

- Completed nodes marked with checkmark on roadmap

### Goal-Task Linking
- Every task links to a goal
- Dashboard shows how today's work maps to long-term plans
- Goal progress updates automatically as linked tasks complete

### Stalled Goal Detection
- If no progress on a goal for 7 days
- ARIA alerts and suggests a micro-task to restart momentum
- "Pick one small thing. Even 5 minutes."

### Project Kanban Board
- To Do / In Progress / Done columns
- Drag to move between columns
- Per-project board
- WIP limits (optional)

## Opportunity Radar

### Runs Daily at 6 AM
- Background agent scans while you sleep (Edge Function or APScheduler)
- Results in morning briefing by 7 AM
- New opportunities count badge on sidebar

### Scans 6 Categories
1. Internships (LinkedIn, Internshala, AngelList)
2. Hackathons (Devfolio, MLH, HackerEarth)
3. Open Source (GitHub Good First Issues, GSoC organisations)
4. Startup Competitions (NASSCOM, Startup India, Atal Innovation Mission)
5. Fellowships (Buddy4Study, Vidyasaarathi, Tata Trusts, international)
6. Freelance demand (Fiverr, Upwork — high demand, low supply matching skills)

### Matching Algorithm
- Skill match score (40% minimum threshold)
- Deadline urgency
- Effort estimation (quick apply > long project)
- Personalization from history
- One-sentence relevance reason

### Critical Alerts
- Opportunity closing in under 48 hours
- Immediate push notification (not waiting for morning briefing)
- In-app red badge on Opportunities nav item

### Learns from You
- After 2 months: knows which types you act on vs skip
- Deprioritizes skipped categories
- Boosts scores for categories you consistently apply to
- Adjusts based on application success rate

### Opportunity Profile Settings
- Opportunity types toggle
- Min match score slider (default 60)
- Location: remote / hybrid / specific city
- Excluded categories multi-select
- Saves to users_profile.opportunity_preferences

### Results Display
- Card grid: title, type badge (color per type), deadline countdown
- Match score bar (green >= 70, yellow 50-69, red < 50)
- Match reason text
- Apply / Save / Dismiss buttons
- Stats card: found this week / applied / best score

## Sleep Monitor

### One-Tap Logging
- Bedtime button: tap when going to sleep
- Wake-up button: tap when waking up
- Duration auto-calculated
- Quality rating (1-5 stars) after wake-up

### Sleep Score
- 0-100 based on duration and quality
- Formula: score = Math.min(100, (duration_minutes/480 * 60) + (quality_rating * 8))
- Score breakdown on hover
- Color: green > 70, yellow 50-69, red < 50

### Task Adjustment
- Low sleep score (<50) shifts heavy cognitive work
- Heavy tasks: coding, system design, competitive programming, new concepts
- Light tasks: review, admin, reading, planning
- Automatic reprioritization in task manager

### Sleep Debt Tracking
- Accumulated debt across days and weeks
- 8 hours = baseline, <8 = debt accrued
- "Sleep debt: 12 hours this week"
- Alert when cumulative debt exceeds 10 hours

### Bedtime Reminder
- Notification at set bedtime (from user profile)
- Shows tomorrow's first task so user sleeps with intention
- Wind-down reminder 30 minutes before bedtime
- Bedtime Agent cron (9:30 PM)

### Weekly Sleep Report
- Average score for the week
- Best and worst days
- Sleep-productivity correlation chart
- "When you sleep >7h, your task completion rate is 2x higher"

### Google Fit Integration
- Auto-import sleep data on Android
- Manual logging fallback
- Data merges with manual logs

## ARIA — Your AI Agent

### What ARIA Knows
- Every course enrolled in and progress on each
- Every skill (have, learning, gaps)
- All active goals and progress
- Every project phase and last action
- All income sources and trends
- Daily task completion rate
- Every idea captured and its status
- Sleep patterns and productivity correlation
- Every opportunity shown and action taken
- Preferences and patterns from all conversations

### What ARIA Does
- Chat with you naturally via floating panel
- Answer questions about your data
- Take actions through conversation (add task, update course, save idea)
- Generate daily briefings at 7 AM
- Create weekly reviews on Sunday
- Suggest what to focus on based on context
- Break down complex tasks into subtasks
- Surface forgotten knowledge at the right time

### 8 AI Sub-Agents
1. Daily Briefing Agent — 7 AM every day
2. Missed Task Checker — every 15 minutes
3. Opportunity Radar — 6 AM every day
4. Roadmap Update Checker — Sunday 9 AM
5. Weekly Review Agent — Sunday 8 PM
6. Bedtime and Sleep Agent — 9:30 PM every night
7. Habit Miss Checker — midnight every day
8. Course Progress Nudge — 6 PM every day

### Communication Model
- Orchestrator receives message first
- Calls relevant agents in parallel where possible
- Combines outputs into coherent reply
- All inter-agent data flows through Supabase

### Memory System
- Each conversation: extract facts, preferences, patterns
- Store in aria_memory with confidence score
- Memory types: preference / fact / pattern / decision
- Low-confidence memories replaced over time
- Context builder injects relevant memories into prompts

### Voice Interaction
- Microphone button in ARIA input
- Web Speech API — free, built into browser
- Speech recognition: English (India)
- Auto-submit after 2 seconds silence
- Pulsing red recording animation

- NetworkFirst for API calls
- CacheFirst for static assets
- IndexedDB for offline action queue
- Install banner after 3rd visit

### Onboarding Wizard
- 5-step flow (only if onboarding_completed=false)
- Step 1: Profile (name, college, year, branch)
- Step 2: Skills (60+ common CS skills, know/learning toggle)
- Step 3: Active courses (name, platform, target completion)

- Step 5: Daily routine (study start, hours, days)
- Triggers first daily briefing immediately on completion

### Data Export
- Export all 21 tables as JSON
- Download: second-brain-export-YYYY-MM-DD.json
- Available from Settings page
- One-click, no waiting

### Mobile Features
- Bottom nav bar on mobile (Home, Tasks, ARIA, Opportunities, Profile)
- Swipe right on task = complete
- Swipe left on task = reschedule
- All interactive elements min height: 44px
- Touch-optimized interactions

### Security Architecture
- RLS on all 21 tables: auth.uid() = user_id
- HTTPS/TLS 1.3 for all data in transit
- AES-256 at rest (Supabase default)
- JWT auto-refresh, 7-day expiry
- Force logout all devices from Settings
- Magic link login option
- Rate limits: 20 AI/min, 10 uploads/hr, 50 searches/day
- All API keys server-side only
- Never in client code or git history
