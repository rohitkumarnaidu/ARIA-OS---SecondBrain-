## Introduction

The Project Tracker helps you manage multi-phase projects. Track progress across phases, log blockers, link to repositories, and keep a clear view of what needs to happen next.

## Creating a Project

1. Go to **Projects** from the sidebar
2. Click **New Project**
3. Fill in the details:

| Field | Description |
|---|---|
| Name | Project name (required) |
| Description | Overview of what the project does |
| Status | Active, On Hold, Completed, Abandoned |
| Phase | Current development phase (Planning, Design, Development, Testing, Deployment, Maintenance) |
| Repository URL | Link to GitHub/GitLab repository |
| Next Action | Single next step to move forward |

## Managing Phases

Projects progress through these phases:

| Phase | Activities |
|---|---|
| Planning | Requirements, architecture, task breakdown |
| Design | Wireframes, mockups, technical design |
| Development | Coding, implementation, feature building |
| Testing | QA, bug fixes, user testing |
| Deployment | Release, CI/CD, monitoring |
| Maintenance | Updates, bug fixes, improvements |

Update the phase field as your project advances. The project card shows the current phase with a progress indicator.

## Tracking Blockers

Blockers are obstacles preventing progress:

1. Open the project detail view
2. Add a blocker with description and priority
3. Mark as resolved when the blocker is cleared
4. Blockers appear as warnings on project cards

## Linking to GitHub

Add your repository URL to:
- Link directly to your code from the project
- Enable quick access to issues and PRs
- Track commits and contributions

## Managing Projects

### Viewing and Filtering

The Projects page provides:
- **Card view**: Visual cards with phase indicators and progress bars
- **List view**: Compact table with sortable columns
- **Filter by status**: Active, on hold, completed, abandoned
- **Filter by phase**: Planning, Design, Development, Testing, Deployment, Maintenance
- **Search**: Find projects by name or description

### Editing Projects

1. Click on any project card to open the detail view
2. Update any field as the project evolves
3. Advance the **Phase** as you move through development stages
4. Add or resolve **Blockers** as issues arise
5. Update the **Next Action** to reflect the current priority

### Project Detail View

The detail view provides a comprehensive project dashboard:
- **Header**: Name, description, status, repository link
- **Phase indicator**: Current phase with progress through the lifecycle
- **Next action**: Single priority task -- the most important thing to do now
- **Blockers**: List of current obstacles with priority and resolution status
- **Linked tasks**: Tasks created from this project's next actions
- **Linked income**: Revenue generated by this project
- **Activity log**: Timeline of phase changes, blocker updates, and status changes
- **Notes**: Free-form notes for project documentation

### Managing Blockers

Blockers track what is preventing progress:

1. Click **Add Blocker** in the project detail view
2. Enter a description of the obstacle
3. Set the **Priority**: low, medium, high, critical
4. When resolved, click **Resolve** with a resolution note
5. Resolved blockers are archived but visible in the activity log

Blocker escalation:
- High/critical blockers appear on the dashboard
- Blockers unresolved for 7+ days trigger a notification
- Project cards show a warning indicator when blockers exist

### Phase Advancement

When advancing a project phase:
1. Update the **Phase** field to the next stage
2. The system logs the phase change in the activity log
3. Linked tasks from the previous phase can be archived
4. New tasks can be generated for the next phase

### Linking Projects to Income

If a project generates revenue:
1. Edit the project and note it in the description
2. On the Income page, create entries linked to this project
3. The project detail shows total linked income
4. This helps you evaluate which projects are most valuable

### Converting Ideas to Projects

When a validated idea is ready to execute:
1. Go to the **Ideas** page
2. Open the validated idea
3. Click **Promote to Project**
4. A new project is created with the idea's details pre-filled
5. The project appears in your project list ready for phase management

## Common Questions

**Q: What if a project is blocked for a long time?**
Set the status to **On Hold** and document the blocker. The project remains visible in your list but does not appear on the dashboard. Resume when the blocker is resolved.

**Q: Should I create a project for every task?**
No. Projects are for multi-phase work with multiple components. Use tasks for single actions. A good rule: if it takes more than a week and has distinct stages, it is a project.

**Q: How do I know when a project is done?**
When all phases are complete and the deliverable is deployed or delivered, set the status to **Completed**. The project remains in your history for reference.

## Tips

- Keep the Next Action field current -- it should be the single task you can do right now
- Update blocker status promptly so the dashboard reflects reality
- Link projects to income entries if they generate revenue
- Convert validated ideas from the Idea Vault into projects
- Limit active projects to 3-4 at a time to maintain focus
- Archive completed projects to keep the active list manageable
- Use the repository link to quickly jump to your code
