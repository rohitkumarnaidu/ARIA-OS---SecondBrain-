## Introduction

The Resource Library is your curated collection of learning materials -- articles, videos, tools, documentation, and references. Save, tag, and organize resources for easy access later.

## Adding a Resource

1. Go to **Resources** from the sidebar
2. Click **Add Resource**
3. Fill in the details:

| Field | Description |
|---|---|
| Title | Name of the resource (required) |
| URL | Link to the article, video, or tool |
| Type | Article, Video, Tool, Documentation, Book, Course, Paper, Other |
| Tags | Comma-separated keywords for categorization |
| Notes | Your summary or key takeaways |
| Status | Unread, Reading, Read, Reference |

## Organizing with Tags

Tags are the primary organization system for resources:
- Add multiple tags per resource (e.g., "python", "machine-learning", "tutorial")
- Filter resources by clicking on any tag
- Use consistent tag names to avoid duplicates
- Tag categories: language, topic, type, difficulty, project

## Searching Resources

- Use the **Search** bar to find resources by title, URL, or notes
- Filter by type using the type dropdown
- Filter by status (unread, reading, read, reference)
- Combine filters for precise results

## Managing Resources

### Viewing and Organizing

The Resources page provides:
- **Grid view**: Visual cards with type icons and thumbnails
- **List view**: Compact table with sortable columns
- **Tag filtering**: Multi-select tag filter with AND/OR toggle
- **Type filter**: Filter by article, video, tool, documentation, book, course, paper, other
- **Status filter**: Filter by unread, reading, read, reference
- **Search**: Full-text search across title, URL, and notes
- **Sort**: By date added, title, type, or status

### Editing Resources

1. Click on any resource to open the detail view
2. Edit the title, URL, type, tags, or notes
3. Update the **Status** as you progress through the material
4. Tags can be added or removed from the tag editor

### Using Tags Effectively

Tags are the primary organization system:
- **Consistency**: Use consistent tag names (e.g., always "python" not "Python" or "py")
- **Granularity**: Use specific tags that enable precise filtering
- **Combinations**: A resource can have multiple tags for multi-dimensional filtering
- **Common categories**: language (python, js, rust), topic (ml, web, db), type (tutorial, ref, tool), difficulty (beginner, intermediate, advanced), project (thesis, side-project)

### Resource Status Workflow

A recommended reading workflow:
1. Save as **Unread** when you discover the resource
2. Change to **Reading** when you start consuming it
3. Change to **Read** when you finish and add notes/summary
4. Change to **Reference** for materials you want to keep for future lookup

### Batch Operations

For managing multiple resources:
- Select multiple resources with checkboxes
- **Batch tag**: Add or remove tags from selected items
- **Batch status**: Change status for multiple items
- **Batch delete**: Remove multiple resources at once

## Tips

- Save resources immediately when you find them -- you can tag and organize later
- Use meaningful tags that match how you search, not how you found the resource
- Write a 1-2 sentence summary in the notes field for quick reference without revisiting
- Periodically clean up resources that are no longer relevant to keep the library focused
- Mark resources as Read once finished to track what you have consumed
- Set a weekly target for clearing unread resources (e.g., read 3 saved articles per week)
- Use the search function when you need to find something specific quickly
